Your Coding Sidecar

Use emprise alongside Claude Code, GitHub Copilot, or any coding assistant. While they focus deep on your codebase, emprise handles everything around it — system monitoring, environment setup, tool management, multi-LLM queries, and quick lookups that don't need your main AI's context window.

Interactive Everything

Arrow-key menus for profiles, tools, models, history, prompts, and config. Consistent UX everywhere. Tab completion for all commands.

43+ Built-in Tools

System info, files, git, shell, web, Docker, SSH, clipboard, dev tools — categorized and toggleable. Async execution keeps the UI responsive.

4 LLM Providers

Ollama (local, free), GitHub Models (free with Copilot), OpenAI (GPT-4o), Anthropic (Claude). Switch instantly with /profile.

Prompt Engineering

Role-based emphasis presets (developer, devops, cloud, data, security, admin). Per-model overrides. Per-tool rules. Test your prompt against the LLM.

Dev Environment Audit

/dev scans 40+ tools across 12 categories. Missing something? Select it and emprise installs it. Platform-aware: brew, apt, or winget.

Diff View

File writes show line-numbered diffs with syntax highlighting via chroma. Green for additions, red for removals, context lines around changes.

Use Cases

Sidecar to Claude Code

Run emprise in a second terminal while Claude Code handles your codebase. Ask emprise for system diagnostics, quick lookups, or a second opinion from a different LLM — without polluting Claude's context window.

New Machine Setup

Type /dev on a fresh machine. emprise audits what's installed, shows what's missing, and installs tools with one keypress. Git, Node, Go, Docker, VS Code, Nerd Font — all platform-aware.

Quick System Check

"What's using my CPU?" — emprise reads the tools first, then explains. No guessing. Real data from get_cpu, get_memory, get_processes, presented in a concise answer.

Multi-LLM Workflow

Ask your local Ollama for quick questions (free, private). Switch to GPT-4o for complex reasoning. Try Claude for code review. All from the same interface with /profile.

Code Generation

Ask emprise to write code — it creates the file directly (never dumps code inline), shows a diff preview with syntax highlighting, and offers to run it.

Prompt Engineering

Configure how emprise talks to the LLM per role (developer, devops, security), per model (small models get simpler instructions), and per tool. Test your prompt configuration against the LLM.

Prompt Engineering System

emprise assembles the system prompt in 6 layers. Each layer adds context and rules that shape how the LLM behaves.

1

Base

Identity, system info, OS, working directory, platform tools

2

Capability Tier

Auto-detected from model name. Small models get simpler instructions.

3

Role Defaults

6 presets: developer, devops, cloud, data, security, admin

4

User Rules

Custom instructions and user name from config

5

Model Overrides

Per-model tool limits, response length, extra rules

6

Emphasis

40+ toggleable rules. High emphasis repeats 3x for weak models.

Example Config

prompts:
  user_name: Steve
  role: developer
  emphasis:
    write_to_file: high
    branch_workflow: medium
    docker_security: off
  model_overrides:
    "llama3.2:3b":
      max_tools_per_turn: 1
      extra_rules:
        - "respond in 1-3 sentences only"

Safety

Blocked Commands

rm -rf /, dd, mkfs and other destructive operations are hard-blocked and will never run.

Protected Paths

/etc, /usr, /System, and home dotfiles are protected from modification.

Inline Visibility

Every tool call is shown with its arguments and output. Nothing runs hidden.

Async + Cancellable

Tools run asynchronously. Press esc or ctrl+c to cancel any operation at any time.

No Exfiltration

Your code is only sent to the LLM provider you configured. Nothing else.

Process Management

/ps shows running processes, /stop kills them. Exit warning if processes are still running.

The Story

emprise is named after Emprise, the first book in Michael P. Kube-McDowell's Trigon Disunity trilogy. In the novel, humanity builds a massive radio telescope to make first contact with an alien civilization — a bold, almost impossible undertaking driven by curiosity and courage.

The word "emprise" itself means "a bold or chivalrous undertaking." That's what building a personal AI assistant feels like — reaching out into the unknown with the tools you have, making something useful from complexity.

The radio dish in the logo represents that telescope — humanity's signal reaching outward. The teal color scheme echoes the cold clarity of space and the warmth of connection.

emprise was built by Steve Sparks as a companion tool for developers who work with AI coding assistants. It started as an experiment in Go with bubbletea, grew into a full TUI with 43+ tools, and evolved into a prompt engineering platform that adapts to how you work.

Built with the help of Claude (Anthropic) through extensive pair programming sessions using Claude Code. The entire codebase, CI/CD pipeline, and this website were developed collaboratively — human vision and AI execution working together. Which is exactly what emprise is designed to enable for you.

FAQ

How is emprise different from Claude Code?

Claude Code is a deep coding assistant — it understands your codebase, writes code, and manages complex refactors. emprise is a broad system companion — it monitors your machine, manages your dev environment, queries multiple LLMs, and handles the operational side of development. They work great together: Claude Code in one terminal for deep coding, emprise in another for everything else.

Do I need an API key?

Not necessarily. You can use Ollama for fully local, free LLM access — no API key, no network, complete privacy. emprise also supports GitHub Models (free with a Copilot subscription), OpenAI (pay-per-token), and Anthropic (pay-per-token).

Is it safe to let emprise run commands?

emprise has multiple safety layers: destructive commands are hard-blocked (rm -rf, dd, mkfs), protected system paths can't be modified, every tool call is shown with its arguments before running, and you can cancel any operation with esc. Tools execute asynchronously so the UI is always responsive.

Can I use it on Linux/Windows?

Yes. emprise is cross-platform Go. The /dev environment audit uses platform-appropriate package managers (brew on macOS, apt on Linux, winget on Windows). All tools work on all platforms.

How does the prompt engineering work?

emprise builds the system prompt in 6 layers: base identity, capability tier (auto-detected from model size), role defaults (developer/devops/etc.), your custom rules, model-specific overrides, and emphasis rules. You configure it through /prompts or in ~/.emprise/config.yaml. High-emphasis rules are repeated multiple times for smaller models that tend to ignore instructions.

Can I use my own Ollama models?

Yes. Run /model to see all models on your Ollama server. emprise queries the Ollama API directly and adapts its prompt based on the model's capability tier.

What's the .emprise.md file?

Drop an .emprise.md file in your project root with notes about your codebase, conventions, or anything the LLM should know. emprise reads it automatically and adds it to the system prompt as project context.

How do I update?

Type "emprise update" — from the command line or inside emprise itself. On macOS, it downloads the signed DMG, replaces the .app bundle, and restarts. emprise also checks for updates at launch and hourly.
